AI Creative on your own machine by The Diffusers of Hugging Face

Last month, big firms published several text-2- image structures: ImageGen by Google, Dall-E2 by OpenAI, and so on. Their core technology is the diffusion model, which Jascha Sohl-Dickstein proposed in 2015. This technique is hugely intriguing, and I am sure I will write an article about it shortly. Even though such text-2-image models are beautiful and show off their incredible results, putting your hand on the tech is still not easy for every people. The problem is that the companies did not publish their models; they use them for a SaaS service and require money for each shot.

Thanks to Huggingface's engineers, they have worked very hard to rebuild the fantastic models of Google, Facebook, OpenAI, and so on and provide them for free to everyone. This generous work is put under the section BigScience. I have tried their publication from last month for mini Dall-E [Link to the trial] hosted on their page. Last week, I was much happier since they published a new library called Diffusers which allows people to create their applications using the diffusion model for free on their machine.

Getting Started

To install the library: pip install diffusers

I have developed source code for you to use with Streamlit. Please check my code on Github.

After installing the code, you can run it on your localhost address with a port 8501. Then, you insert a prompt and receive a generated photo.

Walkthrough the code

Basic streamlit

This is an excellent library for data scientists to serve their machine learning models on types of web-based applications. Streamlit contains several APIs that allow you to make a complete front-end swiftly. You do not need to know Pytkinter, PythonQT, PythonSimpleUI, or any programming language to create your own front-end. Streamlit provided you with several tools that you need. Believe me. Even the research groups at the top universities or The New York Times have used Streamlit to demonstrate their data.

Install by typing pip install streamlit and start using by import streamlit; now, you can begin using Streamlit in your code. Below is my post about a simple code to create a sidebar of one webpage—more instructions are on the Streamlit website.

Model loading

To avoid reloading the generation model for every new running, you must use the below code to cache the model. Streamlit will cache your model on Ram.

@st.cache(hash_funcs={torch.nn.parameter.Parameter: lambda _: None}, allow_output_mutation=True)

I also provide information related to the running speed and memory consumption. For the first shot, I calculated that the model requires about 40 seconds to complete a photo (or an error in my code :)). However, the following operations only take only about 3 or 4 seconds. All my experiences are on a big server with about 48 cores parallel running without GPU support. Memory consumption is about 7GB in total.
